🔍 ¿Por qué validar antes de aplicar?

YAML es estricto: un tab, un espacio mal colocado o un tipo de dato incorrecto rompe el parseo. Aplicar un YAML sin validar puede dejar contenedores huérfanos, exponer puertos o tumbar servicios en producción.

💡 "Validar un YAML cuesta segundos. Depurar un despliegue roto cuesta horas."

Las herramientas de validación actúan como filtros progresivos: primero revisan sintaxis pura, luego estructura, luego lógica específica de la plataforma (Compose o K8s). Nunca saltes pasos.

🧰 Catálogo de Validadores

🔤 yamllint

yamllint archivo.yml

Valida sintaxis YAML pura: indentación, espacios tras :, tabuladores y estilo. No entiende de Docker ni K8s.

🔄 yq

yq -o=json archivo.yml

Convierte YAML a JSON. Si falla aquí, la jerarquía está rota. Ideal para depurar mapas/listas complejas visualmente.

🐳 docker compose config

docker compose config --quiet

Valida lógica de Compose: puertos, redes, volúmenes, variables y ciclo de vida. No arranca contenedores.

☸️ kubectl dry-run

kubectl apply -f manifest.yml --dry-run=client

Valida contra el esquema local de K8s. Detecta campos desconocidos, tipos incorrectos y errores de API sin contactar al clúster.
